    API Reference¶

    Welcome to the ProcureChain API Reference. This section documents all APIs, server actions, AI flows, and utilities.

    Overview¶

    ProcureChain's API consists of:

    • Server Actions: Next.js Server Actions for data mutations
    • AI Flows: Genkit flows for AI-powered features
    • Database Schema: Complete data model
    • Utilities: Helper functions and tools

    Contents¶

    Server Actions¶

    All server actions for data mutations, organized by feature.

    AI Flows¶

    Genkit flows for AI analytics and conversational features.

    Database Schema¶

    Complete database schema with relationships and types.

    Utilities¶

    Helper functions, validators, and utilities.

    Server Actions¶

    Server Actions are the primary way to mutate data in ProcureChain. They run exclusively on the server and provide type-safe data operations.

    Example Usage¶

    import { createRequisition } from '@/app/(app)/requisitions/actions';

// In a Client Component
const handleSubmit = async (formData: FormData) => {
  const result = await createRequisition(formData);

  if (result.success) {
    toast.success('Requisition created');
  } else {
    toast.error(result.error);
  }
};

    Available Actions¶

    • Requisitions: Create, update, approve, reject
    • RFQs: Create, publish, close, award
    • Vendors: Create, update, suspend
    • Contracts: Create, amend, terminate
    • Bids: Submit, score, award
    • Invoices: Create, approve, reject
    • Support: Create ticket, update, close

    AI Flows¶

    AI Flows are Genkit-based functions that provide AI-powered features.

    Example Usage¶

    import { conversationalAnalysis } from '@/ai/flows/conversational';

// In a Server Action
const response = await conversationalAnalysis({
  query: "Which vendors have the highest performance scores?",
  context: {}
});

    Available Flows¶

    • Spend Analysis: Analyze spending patterns
    • Predictive Forecast: Forecast future spend
    • What-If Scenarios: Model strategic decisions
    • Conversational Analysis: Natural language queries

    Database Schema¶

    Core Tables¶

    • users: Internal staff users
    • vendors: External vendor accounts
    • requisitions: Procurement requests
    • rfqs: Requests for quotation
    • bids: Vendor quotations
    • contracts: Vendor agreements
    • purchaseOrders: Purchase orders
    • invoices: Vendor invoices

    Relationships¶

    erDiagram
    users ||--o{ requisitions : creates
    vendors ||--o{ bids : submits
    requisitions ||--o{ rfqs : generates
    rfqs ||--o{ bids : receives
    bids ||--o| contracts : awarded
    contracts ||--o{ purchaseOrders : generates
    purchaseOrders ||--o{ invoices : receives

    Type Safety¶

    All APIs are fully typed with TypeScript:

    // Server Action with typed response
type ActionResponse<T> = {
  success: boolean;
  data?: T;
  error?: string;
};

// AI Flow with typed input/output
type ConversationalInput = {
  query: string;
  context: Record<string, any>;
};

type ConversationalOutput = {
  answer: string;
  sources: string[];
};

    Error Handling¶

    Server Actions¶

    try {
  // Validate input
  const validated = schema.parse(formData);

  // Perform operation
  const result = await db.insert(table).values(validated);

  return { success: true, data: result };
} catch (error) {
  return { success: false, error: error.message };
}

    AI Flows¶

    try {
  const response = await flow(input);
  return response;
} catch (error) {
  console.error('Flow error:', error);
  throw new Error('AI analysis failed');
}

    Authentication¶

    All server actions and AI flows require authentication:

    // Get current user
const user = await getCurrentUser();
if (!user) {
  return { success: false, error: 'Unauthorized' };
}

// Check permissions
if (!can(user.role, 'requisitions:create')) {
  return { success: false, error: 'Forbidden' };
}

    Rate Limiting¶

    AI flows have rate limits:

    • Conversational Analysis: 10 requests/minute
    • Spend Analysis: 5 requests/minute
    • Forecasting: 3 requests/minute

    Validation¶

    All inputs are validated with Zod:

    const requisitionSchema = z.object({
  title: z.string().min(1).max(200),
  description: z.string().min(1),
  category: z.enum(['goods', 'services', 'works']),
  estimatedValue: z.number().positive(),
});
