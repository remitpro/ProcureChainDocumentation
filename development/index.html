    Development¶

    Welcome to the ProcureChain Development Guide. This section covers everything you need to know to develop, test, and contribute to ProcureChain.

    Overview¶

    This guide is for developers who want to:

    • Set up a development environment
    • Understand the codebase
    • Add new features
    • Fix bugs
    • Contribute to the project

    Contents¶

    Environment Setup¶

    Configure your development environment with all necessary tools and dependencies.

    Database Management¶

    Work with the database schema, migrations, and Drizzle ORM.

    Server Actions¶

    Create and use Next.js Server Actions for data mutations.

    AI Flows¶

    Develop AI-powered features using Genkit flows and tools.

    Testing¶

    Write and run tests to ensure code quality.

    Code Style¶

    Follow coding standards and best practices.

    Contributing¶

    Guidelines for contributing to ProcureChain.

    Quick Start for Developers¶

    1. Clone the repository 

      git clone https://github.com/remitpro/ProcureChain
cd procurechain

    2. Install dependencies 

      npm install

    3. Set up environment 

      cp .env.example .env
# Edit .env with your values

    4. Initialize database 

      npm run db:push
npm run db:seed

    5. Start development servers 

      # Terminal 1
npm run dev

# Terminal 2
npm run genkit:dev

    Development Workflow¶

    Creating a Feature¶

    1. Create a feature branch 

      git checkout -b feature/your-feature-name

    2. Develop your feature

    3. Write code
    4. Add tests

    5. Update documentation

    6. Test your changes 

      npm run typecheck
npm test
npm run build

    7. Commit and push 

      git add .
git commit -m "feat: add your feature"
git push origin feature/your-feature-name

    8. Create a pull request

    Fixing a Bug¶

    1. Create a bugfix branch 

      git checkout -b fix/bug-description

    2. Fix the bug

    3. Identify the issue
    4. Write a failing test
    5. Fix the code

    6. Verify the test passes

    7. Follow steps 3-5 from "Creating a Feature"

    Project Structure¶

    procurechain/
 src/
    app/              # Next.js app routes
    components/       # React components
    lib/              # Utilities and configurations
    ai/               # Genkit flows and tools
    firebase/         # Firebase configuration
 drizzle/              # Database migrations
 scripts/              # Utility scripts
 docs/                 # Documentation
 tests/                # Test files

    Key Technologies¶

    • Next.js 15: App Router, Server Components
    • TypeScript: Type-safe development
    • Drizzle ORM: Database toolkit
    • Firebase: Authentication
    • Genkit: AI integration
    • ShadCN/UI: Component library
    • Tailwind CSS: Styling

    Development Tools¶

    Required¶

    • Node.js v20+
    • npm v9+
    • Git
    • Code editor (VS Code recommended)

    Recommended¶

    • VS Code Extensions:
    • ESLint
    • Prettier
    • TypeScript
    • Tailwind CSS IntelliSense
    • Drizzle ORM

    Optional¶

    • Docker Desktop
    • Postman/Insomnia (API testing)
    • pgAdmin (database management)

    Coding Standards¶

    TypeScript¶

    • Use strict mode
    • No any types
    • Proper type definitions
    • JSDoc comments for public APIs

    React¶

    • Functional components
    • Server Components by default
    • Client Components only when needed
    • Proper prop types

    CSS¶

    • Tailwind CSS utilities
    • Component-scoped styles
    • Consistent naming

    Database¶

    • Type-safe queries
    • Proper indexes
    • Meaningful column names
    • Foreign key constraints

    Testing¶

    Unit Tests¶

    npm test

    Type Checking¶

    npm run typecheck

    Build Test¶

    npm run build

    Debugging¶

    Next.js¶

    • Use console.log in Server Components
    • Use browser DevTools for Client Components
    • Check .next directory for build output

    Database¶

    • Use Drizzle Studio: npm run db:studio
    • Check query logs
    • Verify migrations

    AI Flows¶

    • Use Genkit Dev UI
    • Check flow traces
    • Review tool calls

    Common Tasks¶

    Adding a New Page¶

    1. Create file in src/app/(app)/your-page/page.tsx
    2. Add to navigation in src/components/sidebar.tsx
    3. Add route protection in src/middleware.ts if needed

    Creating a Server Action¶

    1. Create file in feature directory: actions.ts
    2. Define action with 'use server'
    3. Add validation with Zod
    4. Implement database logic
    5. Return typed response

    Adding a Database Table¶

    1. Define schema in src/lib/schema.ts
    2. Generate migration: npm run db:generate
    3. Apply migration: npm run db:migrate
    4. Update types

    Creating an AI Flow¶

    1. Create flow in src/ai/flows/
    2. Define tools if needed in src/ai/tools/
    3. Register in src/ai/genkit.ts
    4. Create UI component
    5. Call from Server Action
