    Quick Start¶

    Get ProcureChain running locally in under 10 minutes! This guide will walk you through the fastest path to a working installation.

    Prerequisites Checklist¶

    Before you begin, ensure you have:

    • Node.js v20 or later installed (Download)
    • npm package manager
    • Git for cloning the repository
    • A Neon account (Sign up)
    • A Firebase project (Create one)
    • A Google AI Studio API key (Get one)

    Step 1: Clone the Repository¶

    git clone https://github.com/remitpro/ProcureChain
cd procurechain

    Step 2: Install Dependencies¶

    npm install

    This will install all required packages including Next.js, Drizzle ORM, Firebase, and Genkit.

    Step 3: Configure Environment Variables¶

    Create your environment file:

    cp .env.example .env

    Edit .env and fill in the essential variables:

    .env
    # Database
DATABASE_URL="postgresql://user:password@host/database?sslmode=require"

# Firebase (Staff Portal)
NEXT_PUBLIC_FIREBASE_API_KEY="your-api-key"
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN="your-project.firebaseapp.com"
NEXT_PUBLIC_FIREBASE_PROJECT_ID="your-project-id"
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET="your-project.appspot.com"
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID="123456789"
NEXT_PUBLIC_FIREBASE_APP_ID="1:123456789:web:abc123"

# Firebase Admin (Server-Side)
FIREBASE_SERVICE_ACCOUNT='{"type":"service_account","project_id":"..."}'

# Super Admin
SUPER_ADMIN_EMAIL="admin@yourcompany.com"
SUPER_ADMIN_PASSWORD="your-secure-password"

# Gemini API
GEMINI_API_KEY="your-gemini-api-key"

    Getting Your Credentials

    • DATABASE_URL: Copy from your Neon dashboard
    • Firebase Config: Get from Firebase Console Project Settings General
    • Service Account: Firebase Console Project Settings Service Accounts Generate New Private Key
    • Gemini API Key: Get from Google AI Studio

    Step 4: Set Up the Database¶

    Push the database schema to your Neon database:

    npm run db:push

    This creates all necessary tables and relationships.

    Step 5: Seed Initial Data¶

    Populate the database with sample data:

    npm run db:seed

    This creates:

    • Super admin user
    • Sample vendors
    • Sample contracts
    • Sample requisitions
    • Sample RFQs and bids
    • Compliance documents

    Default Credentials

    The seed script creates vendor accounts with the default password: password123

    Change these passwords in production!

    Step 6: Start the Development Servers¶

    ProcureChain requires two processes running simultaneously:

    Terminal 1: Next.js Application¶

    npm run dev

    The application will be available at http://localhost:9002

    Terminal 2: Genkit AI Service¶

    npm run genkit:dev

    This starts the AI service that powers analytics and conversational insights.

    Pro Tip

    Use a terminal multiplexer like tmux or split terminals to run both processes simultaneously.

    Step 7: Log In¶

    Staff Portal¶

    1. Navigate to http://localhost:9002/login
    2. Log in with your super admin credentials:
    3. Email: The email you set in SUPER_ADMIN_EMAIL
    4. Password: The password you set in SUPER_ADMIN_PASSWORD

    Vendor Portal¶

    1. Navigate to http://localhost:9002/vendor/login
    2. Log in with a seeded vendor account:
    3. Email: vendor1@example.com (or any seeded vendor)
    4. Password: password123

    Troubleshooting¶

    Database Connection Issues¶

    If you see database connection errors:

    1. Verify your DATABASE_URL is correct
    2. Ensure your Neon database is active
    3. Check that your IP is whitelisted in Neon settings

    Firebase Authentication Errors¶

    If authentication fails:

    1. Verify all Firebase environment variables are set
    2. Check that the service account JSON is valid
    3. Ensure Firebase Authentication is enabled in your project

    Genkit Service Not Starting¶

    If the AI service won't start:

    1. Verify your GEMINI_API_KEY is valid
    2. Check that port 3400 (default Genkit port) is available
    3. Review the terminal output for specific errors

    Port Already in Use¶

    If port 9002 is already in use:

    # Use a different port
npm run dev -- -p 3000
