    Installation¶

    This guide provides detailed installation instructions for ProcureChain across different environments and scenarios.

    System Requirements¶

    Minimum Requirements¶

    • Node.js: v20.0.0 or later
    • npm: v9.0.0 or later
    • RAM: 4GB minimum, 8GB recommended
    • Disk Space: 2GB for dependencies and build artifacts
    • OS: macOS, Linux, or Windows (with WSL2)

    Recommended Requirements¶

    • Node.js: v20.11.0 (LTS)
    • npm: v10.0.0 or later
    • RAM: 16GB for development
    • Disk Space: 5GB
    • OS: macOS or Linux for best performance

    Installation Methods¶

    Choose the installation method that best suits your needs:

    Perfect for development and testing.

    Local Development Installation¶

    1. Install Prerequisites¶

    Node.js and npm¶

    # Using Homebrew
brew install node@20

# Verify installation
node --version  # Should be v20.x.x or later
npm --version   # Should be v9.x.x or later

    # Using NodeSource repository
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Verify installation
node --version
npm --version
    1. Download the installer from nodejs.org
    2. Run the installer
    3. Verify installation in PowerShell:
    node --version
npm --version

    Git¶

    brew install git

    sudo apt-get install git

    Download from git-scm.com

    2. Clone the Repository¶

    git clone https://github.com/remitpro/ProcureChain
cd procurechain

    3. Install Dependencies¶

    npm install

    This installs all required packages:

    • Next.js: Web framework
    • Drizzle ORM: Database toolkit
    • Firebase: Authentication
    • Genkit: AI integration
    • ShadCN/UI: UI components
    • And many more...

    Installation Time

    Initial installation may take 2-5 minutes depending on your internet connection.

    4. Verify Installation¶

    # Check if all dependencies are installed
npm list --depth=0

# Run type checking
npm run typecheck

    Docker Compose Installation¶

    1. Install Docker¶

    Download and install Docker Desktop for Mac

    # Install Docker Engine
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Install Docker Compose
sudo apt-get install docker-compose-plugin

    Download and install Docker Desktop for Windows

    2. Clone and Configure¶

    git clone https://github.com/remitpro/ProcureChain
cd procurechain
cp .env.example .env

    Edit .env with your production values (see Configuration).

    3. Build and Start¶

    # Build the Docker image
npm run docker:build

# Start all services
npm run docker:up

    This starts:

    • Next.js Application: Port 3000
    • PostgreSQL Database: Port 5432
    • Caddy Reverse Proxy: Ports 80 and 443

    4. Verify Docker Installation¶

    # Check running containers
docker ps

# View logs
npm run docker:logs

# Access the application
# Navigate to http://localhost or your configured domain

    Firebase App Hosting Installation¶

    1. Install Firebase CLI¶

    npm install -g firebase-tools

    2. Authenticate¶

    firebase login

    3. Initialize Firebase¶

    # In your project directory
firebase init apphosting

    Follow the prompts to:

    • Select your Firebase project
    • Choose the backend location
    • Configure build settings

    4. Configure Secrets¶

    In the Firebase Console:

    1. Navigate to App Hosting Your Backend Settings
    2. Add the following secrets:
    3. DATABASE_URL
    4. FIREBASE_SERVICE_ACCOUNT
    5. VENDOR_FIREBASE_SERVICE_ACCOUNT
    6. SUPER_ADMIN_EMAIL
    7. SUPER_ADMIN_PASSWORD
    8. GEMINI_API_KEY

    5. Deploy¶

    firebase apphosting:backends:deploy

    Database Setup

    Firebase App Hosting doesn't automatically run database migrations. You must manually run:

    DATABASE_URL=your_prod_url npm run db:push

    Post-Installation Steps¶

    1. Database Setup¶

    Push Schema¶

    npm run db:push

    This creates all database tables and relationships.

    Seed Data (Optional)¶

    npm run db:seed

    Production Warning

    Only run the seed script in development environments. It will create test data and may overwrite existing data.

    2. Verify Environment Variables¶

    # Check that all required variables are set
node -e "
const required = [
  'DATABASE_URL',
  'NEXT_PUBLIC_FIREBASE_API_KEY',
  'FIREBASE_SERVICE_ACCOUNT',
  'SUPER_ADMIN_EMAIL',
  'GEMINI_API_KEY'
];
const missing = required.filter(v => !process.env[v]);
if (missing.length) {
  console.error('Missing variables:', missing);
  process.exit(1);
}
console.log(' All required variables are set');
"

    3. Build for Production¶

    npm run build

    This:

    • Compiles TypeScript
    • Optimizes assets
    • Generates static pages
    • Creates production bundles

    4. Start Production Server¶

    npm start

    The application will be available at http://localhost:3000 (or your configured port).

    Updating ProcureChain¶

    Update Dependencies¶

    # Update all dependencies to latest compatible versions
npm update

# Or update to latest versions (may include breaking changes)
npm install <package>@latest

    Pull Latest Changes¶

    git pull origin main
npm install
npm run db:push  # Apply any schema changes
npm run build

    Uninstallation¶

    Local Installation¶

    # Remove node_modules
rm -rf node_modules

# Remove build artifacts
rm -rf .next

# Remove the project directory
cd ..
rm -rf procurechain

    Docker Installation¶

    # Stop and remove containers
npm run docker:down

# Remove images
docker rmi procurechain-nextjs-app

# Remove volumes (WARNING: This deletes database data)
docker volume rm procurechain_postgres_data

    Troubleshooting¶

    Node Version Issues¶

    If you encounter Node version errors:

    # Check your Node version
node --version

# Install nvm (Node Version Manager)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Install and use Node 20
nvm install 20
nvm use 20

    Dependency Installation Failures¶

    If npm install fails:

    # Clear npm cache
npm cache clean --force

# Remove node_modules and package-lock.json
rm -rf node_modules package-lock.json

# Reinstall
npm install

    Build Failures¶

    If the build fails:

    # Check for TypeScript errors
npm run typecheck

# Clear Next.js cache
rm -rf .next

# Rebuild
npm run build

    Docker Issues¶

    If Docker containers won't start:

    # Check Docker is running
docker --version

# View detailed logs
docker-compose logs -f

# Restart Docker Desktop (macOS/Windows)
# Or restart Docker service (Linux)
sudo systemctl restart docker
