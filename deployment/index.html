    Deployment¶

    Welcome to the ProcureChain Deployment Guide. This section covers deploying ProcureChain to production environments.

    Deployment Options¶

    ProcureChain supports multiple deployment methods:

    Docker Compose¶

    Deploy using Docker containers with PostgreSQL, Caddy, and optional management tools.

    Best for: Self-hosted deployments, VPS, on-premise

    Firebase App Hosting¶

    Serverless deployment on Google Cloud infrastructure.

    Best for: Scalable cloud deployments, minimal ops overhead

    Quick Deployment¶

    Docker Compose (Recommended)¶

    # 1. Clone and configure
git clone <repository-url>
cd procurechain
cp .env.example .env
# Edit .env with production values

# 2. Build and deploy
npm run docker:build
npm run docker:up

# 3. Initialize database
docker exec -it procurechain-nextjs-app npm run db:push

    Your application is now running at http://localhost (or your configured domain).

    Firebase App Hosting¶

    # 1. Install Firebase CLI
npm install -g firebase-tools

# 2. Login and initialize
firebase login
firebase init apphosting

# 3. Configure secrets in Firebase Console
# Add: DATABASE_URL, FIREBASE_SERVICE_ACCOUNT, etc.

# 4. Deploy
firebase apphosting:backends:deploy

    Pre-Deployment Checklist¶

    Before deploying to production:

    Security¶

    • Strong SUPER_ADMIN_PASSWORD set
    • Unique NEXTAUTH_SECRET generated
    • Separate Firebase projects for staff/vendors
    • SSL/TLS certificates configured
    • Firewall rules configured
    • Database access restricted
    • Environment variables secured

    Configuration¶

    • Production DATABASE_URL set
    • All Firebase credentials configured
    • GEMINI_API_KEY set
    • Domain names configured
    • Email SMTP configured (if applicable)
    • Error tracking configured (Sentry)

    Database¶

    • Database created and accessible
    • Connection pooling enabled
    • Backups configured
    • Schema applied (db:push or db:migrate)
    • Initial data seeded (if needed)

    Application¶

    • Build succeeds: npm run build
    • Type checking passes: npm run typecheck
    • Tests pass: npm test
    • Environment variables validated

    Infrastructure¶

    • Sufficient server resources (CPU, RAM)
    • Disk space available
    • Network connectivity verified
    • DNS records configured
    • Load balancer configured (if applicable)

    Environment Variables¶

    See Environment Variables for complete configuration.

    Required for Production¶

    # Database
DATABASE_URL="postgresql://..."

# Firebase (Staff)
NEXT_PUBLIC_FIREBASE_API_KEY="..."
FIREBASE_SERVICE_ACCOUNT='{"type":"service_account",...}'

# Firebase (Vendor)
NEXT_PUBLIC_VENDOR_FIREBASE_API_KEY="..."
VENDOR_FIREBASE_SERVICE_ACCOUNT='{"type":"service_account",...}'

# Admin
SUPER_ADMIN_EMAIL="admin@yourcompany.com"
SUPER_ADMIN_PASSWORD="strong-password"

# AI
GEMINI_API_KEY="..."

# Security
NEXTAUTH_SECRET="generated-secret"

    Database Migration¶

    Initial Setup¶

    # Push schema to production database
DATABASE_URL=your_prod_url npm run db:push

    Ongoing Migrations¶

    # Generate migration
npm run db:generate

# Review migration files in drizzle/

# Apply migration
DATABASE_URL=your_prod_url npm run db:migrate

    Monitoring¶

    Application Monitoring¶

    • Logs: Check application logs regularly
    • Errors: Configure Sentry for error tracking
    • Performance: Monitor response times
    • Uptime: Set up uptime monitoring

    Database Monitoring¶

    • Connections: Monitor connection pool usage
    • Queries: Track slow queries
    • Storage: Monitor disk usage
    • Backups: Verify backup success

    Infrastructure Monitoring¶

    • CPU: Track CPU usage
    • Memory: Monitor RAM usage
    • Disk: Watch disk space
    • Network: Monitor bandwidth

    Scaling¶

    Vertical Scaling¶

    Increase server resources:

    • More CPU cores
    • Additional RAM
    • Faster disk (SSD)

    Horizontal Scaling¶

    Add more instances:

    1. Deploy multiple Next.js instances
    2. Configure load balancer
    3. Use shared session storage
    4. Enable database connection pooling

    Database Scaling¶

    • Enable connection pooling
    • Add read replicas
    • Optimize queries
    • Add indexes

    Backup & Recovery¶

    Database Backups¶

    # Manual backup
pg_dump $DATABASE_URL > backup.sql

# Restore
psql $DATABASE_URL < backup.sql

    Automated Backups¶

    • Neon: Automatic backups included
    • Self-hosted: Configure cron job
    • Retention: Keep 30 days minimum

    Disaster Recovery¶

    1. Document recovery procedures
    2. Test recovery process regularly
    3. Store backups off-site
    4. Maintain runbooks

    Troubleshooting¶

    Application Won't Start¶

    1. Check environment variables
    2. Verify database connectivity
    3. Review application logs
    4. Check port availability

    Database Connection Issues¶

    1. Verify DATABASE_URL
    2. Check firewall rules
    3. Verify database is running
    4. Check connection limits

    SSL/TLS Issues¶

    1. Verify certificates are valid
    2. Check domain configuration
    3. Review Caddy logs
    4. Ensure ports 80/443 are open

    Rollback Procedures¶

    Application Rollback¶

    # Docker
docker-compose down
git checkout previous-version
npm run docker:build
npm run docker:up

# Firebase
firebase apphosting:rollbacks:create

    Database Rollback¶

    # Restore from backup
psql $DATABASE_URL < backup.sql

    Security Best Practices¶

    1. Use HTTPS: Always use SSL/TLS
    2. Secure Secrets: Never commit secrets to git
    3. Regular Updates: Keep dependencies updated
    4. Access Control: Restrict database access
    5. Monitoring: Enable error tracking
    6. Backups: Regular automated backups
    7. Firewall: Configure firewall rules
    8. Audit Logs: Review logs regularly

    Performance Optimization¶

    1. Enable Caching: Use Next.js caching
    2. Database Indexes: Add appropriate indexes
    3. Connection Pooling: Enable for database
    4. CDN: Use CDN for static assets
    5. Compression: Enable gzip/brotli
    6. Image Optimization: Use Next.js Image component

    Cost Optimization¶

    Firebase App Hosting¶

    • Monitor usage in Google Cloud Console
    • Set up billing alerts
    • Optimize cold starts
    • Use appropriate instance sizes

    Self-Hosted¶

    • Right-size server resources
    • Use reserved instances (cloud)
    • Optimize database queries
    • Implement caching
