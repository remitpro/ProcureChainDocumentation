    Frequently Asked Questions¶

    Find answers to common questions about ProcureChain.

    General Questions¶

    What is ProcureChain?¶

    ProcureChain is a comprehensive, enterprise-grade procurement and vendor management system. It streamlines the entire procurement lifecycle from requisition to payment, with AI-powered analytics and dual portals for staff and vendors.

    Who is ProcureChain for?¶

    ProcureChain is designed for:

    • Enterprises: Large organizations with complex procurement needs
    • Government: Public sector organizations requiring transparent procurement
    • SMEs: Small to medium enterprises looking to streamline procurement
    • Vendors: Suppliers who want to participate in electronic procurement

    Is ProcureChain open source?¶

    ProcureChain is proprietary software. Please contact us for licensing information.

    What are the system requirements?¶

    • Server: Node.js v20+, 4GB RAM minimum
    • Database: PostgreSQL (Neon recommended)
    • Browser: Modern browsers (Chrome, Firefox, Safari, Edge)
    • Mobile: Responsive design works on tablets and phones

    Installation & Setup¶

    How long does installation take?¶

    With all prerequisites ready, installation takes about 10 minutes following the Quick Start Guide.

    Do I need separate Firebase projects?¶

    For development, you can use a single Firebase project. For production, we strongly recommend separate projects for staff and vendor portals for better security and isolation.

    Can I use a different database?¶

    ProcureChain is designed for PostgreSQL. While Drizzle ORM supports other databases, the schema and features are optimized for PostgreSQL. Migration to other databases would require code changes.

    Do I need Docker?¶

    No, Docker is optional. You can run ProcureChain:

    • Locally: With Node.js and npm
    • Docker: Using Docker Compose
    • Firebase: Using Firebase App Hosting
    • Custom: Any Node.js hosting environment

    What if I don't have a Neon account?¶

    Neon is recommended but not required. You can use any PostgreSQL database. Just update the DATABASE_URL in your .env file.

    Features & Functionality¶

    Can vendors register themselves?¶

    Yes! Vendors can self-register through the vendor portal. Procurement staff can then review and approve their accounts.

    How does the approval workflow work?¶

    Requisitions go through a multi-stage approval process:

    1. Created: Initial submission
    2. Pending: Awaiting approval
    3. Approved: Ready for sourcing
    4. Rejected: Sent back for revision

    Approvers are assigned based on roles and permissions.

    Can I customize contract templates?¶

    Yes! Contract templates use Handlebars syntax. You can:

    • Create custom templates
    • Use dynamic variables (vendor name, dates, amounts)
    • Include conditional logic
    • Format text and tables

    How does AI analytics work?¶

    AI analytics uses Google's Gemini models through Genkit:

    • Spend Analysis: Analyzes historical spend data
    • Forecasting: Predicts future spend using ML
    • What-If Scenarios: Simulates different scenarios
    • Conversational: Natural language queries

    The AI has access to your real-time data through secure tools.

    Can I export data?¶

    Yes, you can export:

    • Reports to PDF/Excel
    • Contract documents
    • Vendor information
    • Spend analytics

    Additional export features can be added as needed.

    Does it support multiple currencies?¶

    Currently, ProcureChain uses a single currency. Multi-currency support can be added as a custom feature.

    Can I integrate with other systems?¶

    ProcureChain can be integrated with:

    • ERP Systems: Through API endpoints
    • Accounting Software: Export data for import
    • Email Systems: SMTP for notifications
    • SSO Providers: Firebase supports OAuth

    Custom integrations can be developed.

    Security & Compliance¶

    How is data secured?¶

    ProcureChain implements multiple security layers:

    • Authentication: Firebase Auth with session cookies
    • Authorization: Role-based access control (RBAC)
    • Encryption: HTTPS for all communications
    • Data Validation: Zod schemas for all inputs
    • Audit Logging: Comprehensive activity tracking
    • Soft Deletes: Data retention for compliance

    Is ProcureChain GDPR compliant?¶

    ProcureChain includes features to support GDPR compliance:

    • User data export
    • Data deletion capabilities
    • Audit trails
    • Access controls

    However, compliance also depends on how you configure and use the system.

    Can I control user permissions?¶

    Yes! ProcureChain has a robust RBAC system with roles:

    • Admin: Full system access
    • Manager: Approve requisitions, manage vendors
    • Staff: Create requisitions, view data
    • Vendor: Access vendor portal only

    Permissions can be customized per role.

    How are passwords stored?¶

    Passwords are managed by Firebase Authentication, which uses industry-standard security practices including bcrypt hashing and salting.

    What about audit trails?¶

    ProcureChain maintains comprehensive audit logs for:

    • Contract changes
    • Support ticket updates
    • Requisition approvals
    • Bid submissions
    • User actions

    All logs include timestamps and actor information.

    Performance & Scalability¶

    How many users can ProcureChain support?¶

    ProcureChain has been tested with:

    • 1000+ concurrent users
    • 100k+ database records
    • 100+ requests per second

    Actual capacity depends on your infrastructure.

    Will it work with large datasets?¶

    Yes! The database is optimized with:

    • Proper indexes
    • Efficient queries
    • Pagination for large lists
    • Lazy loading

    Can I scale horizontally?¶

    Yes, ProcureChain can be scaled horizontally:

    • Multiple Next.js instances behind a load balancer
    • Neon PostgreSQL auto-scales
    • Stateless architecture (session in cookies)

    What about caching?¶

    Next.js provides built-in caching:

    • Server Components: Cached by default
    • API Routes: Can be cached with headers
    • Static Assets: CDN caching

    Additional caching can be added with Redis.

    Deployment & Hosting¶

    Where can I deploy ProcureChain?¶

    ProcureChain can be deployed to:

    • Firebase App Hosting: Recommended serverless option
    • Docker: Any Docker-compatible host
    • VPS: Ubuntu, Debian, etc.
    • Cloud: AWS, GCP, Azure
    • On-Premise: Your own servers

    Do I need a domain name?¶

    For production, yes. For development, localhost works fine.

    How do I set up SSL?¶

    • Caddy: Automatic SSL with Let's Encrypt
    • Firebase: SSL included automatically
    • Manual: Use Nginx with Certbot

    What about backups?¶

    Implement backups for:

    • Database: Neon has automatic backups, or use pg_dump
    • Files: If storing uploads, backup storage
    • Configuration: Keep .env backed up securely

    Can I use a CDN?¶

    Yes! Static assets can be served through a CDN. Configure in next.config.ts:

    assetPrefix: 'https://cdn.yourapp.com'

    Troubleshooting¶

    The application won't start¶

    Check:

    1. Node.js version (must be v20+)
    2. All dependencies installed (npm install)
    3. Environment variables set correctly
    4. Database is accessible
    5. Ports 9002 and 3400 are available

    Database connection fails¶

    Verify:

    1. DATABASE_URL is correct
    2. Database is running and accessible
    3. IP is whitelisted (if using Neon)
    4. SSL mode is set correctly

    Firebase authentication errors¶

    Check:

    1. All Firebase environment variables are set
    2. Service account JSON is valid
    3. Domain is authorized in Firebase Console
    4. Firebase Authentication is enabled

    AI features not working¶

    Ensure:

    1. GEMINI_API_KEY is valid
    2. Genkit service is running (npm run genkit:dev)
    3. API quotas not exceeded
    4. Required APIs are enabled in Google Cloud

    Build fails¶

    Try:

    1. Clear cache: rm -rf .next
    2. Reinstall dependencies: rm -rf node_modules && npm install
    3. Check TypeScript errors: npm run typecheck
    4. Review build logs for specific errors
